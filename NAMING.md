# Naming

Last updated: 2026-04-26

This file defines the canonical naming convention for the stack so docs, issues, boards, and agents use the same language.

## Canonical Names

- System / project name: `bankst-os`
- Public frontend / app domain: `bankst.co`
- Frontend repo/runtime name: `bankst-os-frontend`

## Usage Rules

- Use `bankst-os` when referring to the overall platform or project.
- Use `bankst.co` when referring to the public-facing frontend/app domain.
- Use `bankst-os-frontend` when referring specifically to the frontend repo or frontend runtime.

## Avoid

- Do not introduce alternate umbrella names unless they are explicitly documented here.
- Do not use a local clone name as the product/system name.

## Examples

- Correct: `bankst-os Cross-Repo Architecture`
- Correct: `bankst.co is the public frontend domain`
- Correct: `bankst-os-frontend is the authoritative frontend repo on the Mac`
## Notes For Agents

- If a new doc, board, issue, or handoff note needs a top-level name, default to `bankst-os`.
- If public exposure or user-facing app access is being described, use `bankst.co`.
