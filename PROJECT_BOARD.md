## Operating Rules

- Projects should be outcome-based, not task dumps.
- Each project should have one owner (the accountable human, not an agent).
- Close a project only after the system behavior is verified, not just after code lands.
- Cross-repo or system-level risks that need a project should be logged in `ISSUE_REGISTRY.md` first, then promoted to a Linear project.
