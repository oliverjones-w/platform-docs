# Task Patterns

Reusable patterns promoted from agent completion reports.

## What belongs here

- Confirmed step-by-step procedures for recurring task types
- Patterns validated across 2+ agent completion reports
- Anything too operational for DECISIONS/ but too reusable for a one-off RUNBOOK

## What does not belong here

- One-off debug notes (those belong in GitHub issue comments)
- In-progress investigations
- Decisions about architecture (those belong in DECISIONS/)
- Recovery procedures for known failures (those belong in RUNBOOKS/)

## Promotion path

Agent completion report → human triage (`board.sh review`) → if pattern is reusable → add here
